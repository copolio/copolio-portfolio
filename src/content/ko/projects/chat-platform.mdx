## 프로젝트 개요

WebSocket 기반의 실시간 채팅 플랫폼을 개인 프로젝트로 개발했습니다. 대규모 동시 접속 환경에서도 안정적인 메시지 전달을 목표로 설계했습니다.

## 주요 업무

### 서버 아키텍처
- NestJS 기반 WebSocket 게이트웨이 설계 및 구현
- Redis Pub/Sub을 활용한 다중 서버 인스턴스 간 메시지 동기화
- MongoDB를 사용한 채팅 이력 저장 및 페이지네이션 API 구현

### 클라이언트 개발
- TypeScript 기반 React 클라이언트 애플리케이션 개발
- 낙관적 업데이트(Optimistic Update) 패턴 적용으로 UX 개선
- 오프라인 메시지 큐잉 및 재연결 시 자동 동기화 구현

### 인프라
- Docker Compose를 활용한 로컬 개발 환경 구성
- k6를 활용한 부하 테스트 시나리오 작성 및 성능 측정

## 성과

- 동시 접속자 1만명 지원 달성 (k6 부하 테스트 검증)
- 메시지 전송 지연시간 50ms 이하 달성
- WebSocket 연결 안정성 99.5% 달성 (24시간 연속 테스트 기준)

## 기술적 도전과 해결

### 대규모 동시 접속 처리
- **문제**: 단일 NestJS 인스턴스에서 5천 동시 접속 초과 시 성능 저하
- **해결**: Redis Pub/Sub 기반 수평 확장 아키텍처 적용, Sticky Session을 통한 연결 분산

### 메시지 순서 보장
- **문제**: 다중 서버 환경에서 메시지 순서가 보장되지 않는 문제
- **해결**: Snowflake ID 기반 메시지 정렬 및 서버 타임스탬프 동기화 적용
